When HTML5 has become a standard?What were the main reasons for developing new standard?

HTML5 became a standard on October 28, 2014, when the World Wide Web Consortium published its Recommendation of HTML5. The main reasons for developing this new standard were to introduce advancements in web technologies, improve mobile-friendliness, provide semantic markup for better structure, establish standardization and interoperability, and promote innovation and market participation. HTML5 aimed to address the evolving needs of web development, improve user experiences, and provide a modern and standardized platform for building web content and applications.

What is a difference between W3C HTML5 and WHATWG HTML Living Standard?

In 2011 the groups came to the conclusion that they had different goals: the W3C wanted to publish a "finished" version of "HTML5", while the WHATWG wanted to continue working on a Living Standard for HTML, continuously maintaining the specification rather than freezing it in a state with known problems, and adding new features as needed to evolve the platform.

Comment on semanticand presentational markup. Why presentational markup is considered deprecated now?

Semantic markup refers to using HTML elements that convey meaning and structure to the content, making it more accessible and understandable to both humans and machines. Semantic markup focuses on accurately representing the purpose and meaning of the content, such as using ◁header▷ for a page header or ◁nav▷ for navigation. On the other hand, presentational markup involves using HTML elements primarily for visual styling purposes, without considering the meaning or structure of the content. Examples of presentational markup include using ◁b▷ for bold text or ◁font▷ for changing font styles. Presentational markup is considered deprecated now because it separates the visual presentation from the actual content, making it harder for assistive technologies and search engines to interpret and navigate the page. It also creates maintenance issues since presentational styles are better handled through CSS. Semantic markup, on the other hand, improves accessibility, search engine optimization, and overall code maintainability by providing a clear structure and meaning to the content.

How you can check browsersupport for particular tag or attribute?

You can check the browser's support for a specific tag or attribute using browser compatibility tables such as MDN Web Docs or Can I use, or check the official documentation of browser manufacturers. You can also use feature detection using the Modernizr JavaScript library to determine whether HTML functions are supported in the user's browser.

What are the audio and video formats, officially supported byHTML standard? What happens if you use other format?

The HTML standard does not specify a specific set of audio and video formats that are officially supported. Different browsers have their own preferred formats, and the support for specific formats may vary. However, commonly supported video formats include MP4, WebM, and Ogg, while commonly supported audio formats include MP3, Ogg, and WAV. If you use a format that is not supported by a particular browser, the browser will not be able to play the media file, so it's important to consider the compatibility of the formats you choose to ensure broad support across different browsers and devices.
